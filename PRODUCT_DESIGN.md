# 微拟课管家产品设计方案

## 1. 产品定位

微拟课管家面向个人家教老师、小型家教工作室和轻量教培团队，核心目标是把排课、作业、题库、上课记录、家长确认和收费对账放在一个系统里。

第一阶段建议先做网站 H5 + 微信小程序共用后端。老师端优先做网页和小程序双端，家长与学生端优先做微信小程序，因为家长通知、确认课时、查看作业更适合微信场景。

## 2. 面向人群

### 家教老师

- 需要快速排课、调课、记录课时。
- 需要给不同学生或同一节课学生布置不同作业。
- 需要沉淀自己的例题、讲义、错题和课程模板。
- 需要月底和家长对账，降低沟通成本。

### 学生

- 查看自己的课程时间、作业、批改反馈、错题本。
- 提交作业图片、文件、语音说明。
- 收藏题型和复习老师标记的重点。

### 家长

- 查看孩子什么时候上课、有没有请假或调课。
- 确认课时、查看账单和付款记录。
- 查看作业完成情况、课堂反馈和阶段报告。

### 助教或合作老师

- 参与批改、上传题目、维护班级。
- 只能看到被授权的学生、班级和课程。

### 平台管理员

- 面向 SaaS 商业化，管理老师租户、套餐、模板、付费状态和数据概览。

## 3. 核心页面设计

### 3.1 老师工作台

- 今日课程数量、待批作业、当月课时、待确认费用。
- 今日课程时间线：上课时间、学生或班级、课程主题、签到状态、家长确认状态。
- 待办事项：待发布作业、待批改、待确认课时、待发送账单。
- 快捷入口：新建课程、上传作业、发布题型、生成账单。

### 3.2 排课页面

- 周视图、月视图、按学生、按老师、按教室或线上会议筛选。
- 支持一对一、小班课、试听课、补课、临时调课。
- 课程字段：学科、学生或班级、开始时间、结束时间、课时数、单价、上课方式、课程主题。
- 冲突检测：学生冲突、老师冲突、休息时间冲突、同一学生连续课提醒。
- 通知能力：调课后自动推送给学生和家长。
- 课程状态：待上课、已签到、已完成、请假、补课中、已取消。

### 3.3 作业页面

- 老师上传作业文件、图片、题目文本或从题库选择题目。
- 可见范围支持单个学生、同一节课学生、指定班级、公开题库。
- 支持截止时间、补交设置、提醒设置、分层作业。
- 学生提交图片、PDF、语音或文本。
- 老师批改：圈画、文字点评、得分、错因标签、优秀作业展示。
- 自动生成错题本和讲评清单。

### 3.4 典型例题论坛

- 老师和学生都可以上传觉得好的题型，但发布权限需要分级。
- 分类：学科、年级、知识点、题型、难度、考试地区。
- 支持收藏、点赞、评论、变式题、讲解视频或图文解析。
- 私有题库、班级题库、公开题库三种范围。
- 老师可把论坛题目一键转为作业、讲义或课堂例题。

### 3.5 上课记录与家长确认

- 每节课生成上课记录：时间、课时、主题、学生表现、作业完成情况、下次计划。
- 家长端确认课时，确认后进入账单。
- 异常记录：请假、迟到、延时、未上课、补课。
- 阶段反馈：按周或月生成学习报告。

### 3.6 收费与账单页面

- 按学生、班级、月份统计课时。
- 支持课时包、按次收费、按月收费、小班按人收费。
- 家长确认后生成账单，可导出 PDF 或图片。
- 支持欠费提醒、预收余额、优惠、补课抵扣。
- 老师可看到收入趋势、未确认金额、已收款金额。

### 3.7 学员与班级页面

- 学生档案：年级、学校、学科、家长联系方式、价格、剩余课时、薄弱点。
- 班级档案：学生名单、固定上课时间、统一价格、课程计划。
- 成长记录：作业完成率、错题标签、阶段成绩、课堂评价。

### 3.8 老师模板与商业化页面

- 老师可选择行业模板：数学、英语、物理、编程、钢琴、美术等。
- 支持个人品牌主页：介绍、课程价格、试听预约、二维码。
- 支持多租户：每个老师拥有独立空间、独立数据、独立家长入口。
- 平台可售卖套餐：免费版、专业版、工作室版。

## 4. 建议补充功能

- 微信提醒：上课前提醒、作业截止提醒、账单确认提醒。
- 试听课转化：记录试听来源、试听反馈、是否报名。
- 课程素材库：讲义、课件、试卷、视频统一管理。
- AI 辅助出题：按知识点、难度和学生错题生成作业。
- AI 批改辅助：识别图片作业，先给老师一版批改建议。
- 错题标签体系：计算、审题、概念、步骤、书写、模型。
- 数据看板：学生出勤率、作业完成率、续费风险、收入趋势。
- 家长评价与转介绍：满意度、推荐码、老带新优惠。
- 老师协作：一个主老师可邀请助教批作业或代课。
- 数据导入导出：Excel 导入学生、导出课时账单。

## 5. 第一版 MVP 范围

建议第一版只做 5 条主链路：

1. 学生和班级管理。
2. 排课与调课。
3. 作业发布、提交和批改记录。
4. 上课记录与家长确认。
5. 月度课时账单。

题库论坛和 AI 能力可以先做轻量版，作为第二阶段增强。

## 6. 角色权限

| 角色 | 能看到 | 能操作 |
| --- | --- | --- |
| 老师 | 自己全部学生、班级、课程、账单 | 全部管理 |
| 助教 | 被授权班级和学生 | 批改、记录、上传资料 |
| 学生 | 自己的课程、作业、错题、反馈 | 提交作业、收藏题目 |
| 家长 | 孩子的课程、作业状态、课时账单 | 确认课时、查看账单 |
| 平台管理员 | 老师租户、套餐、系统数据 | 套餐与运营管理 |

## 7. 技术建议

### 前端

- 老师后台：React 或 Vue，适合复杂管理界面。
- 微信小程序：Taro、uni-app 或原生小程序。
- H5 家长端：可复用小程序页面逻辑。

### 后端

- Java Spring Boot 适合你现在的技术背景。
- MySQL 存储核心业务数据。
- Redis 做通知、验证码、缓存和任务队列。
- 对象存储保存作业图片、PDF、讲义、视频。
- 微信小程序订阅消息用于提醒。

### 数据表核心

- teacher：老师与租户信息。
- student：学生档案。
- parent：家长账号。
- class_group：班级。
- lesson：课程安排。
- attendance_record：上课与签到记录。
- homework：作业。
- homework_submission：作业提交。
- question：题库题目。
- question_collection：收藏与引用。
- billing_record：课时账单。
- payment_record：收款记录。

## 8. 商业化建议

- 免费版：最多 10 名学生、基础排课、基础账单。
- 专业版：更多学生、作业批改、题库、家长确认、导出账单。
- 工作室版：多老师、多助教、数据看板、品牌主页。
- 增值服务：AI 出题、AI 批改、题库会员、短信通知包。

## 9. 当前工程

当前仓库已经从静态 HTML 原型升级为前后端分离思路下的前端工程：

- `apps/web`：React + TypeScript + Vite Web 管理端。
- `apps/miniprogram`：微信小程序原生端骨架。
- `docs/ARCHITECTURE.md`：Java 21 + Spring Boot 3 + MySQL 的后端架构建议。

本地运行 Web 端：

```bash
npm install
npm run dev
```

老师电脑端优先在 `apps/web` 继续迭代，学生和家长端优先在 `apps/miniprogram` 继续迭代。
